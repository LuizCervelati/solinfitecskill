  • Handler para NotFoundException → 404 com mensagem legível

  • Testar: buscar id que não existe → retorna JSON de erro correto
  • Testar: POST com campo vazio → retorna lista dos campos inválidos
Por que isso importa
⚠ Ponto de atenção Sem @ControllerAdvice, sua API retorna stack trace de Java para o cliente — péssima prática. Recrutadores olham isso. Todo projeto Spring profissional tem essa camada.
✓ Entrega API nunca expõe stack trace. Todos os erros retornam JSON estruturado com status HTTP correto.
